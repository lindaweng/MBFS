My Experience



I learned about testing with Cucumber and Gherkin. Gherkin is just syntax for english language and Cucumber translates that to Java for you (but you have to write Java code once to tell Cucumber what to do for each line of Gherkin). We talked about why they test (to find mistakes, duh).

We gave a presentation on Continuous Integration and Continuous Deployment (CI/CD). We first learned what CI/CD was from a couple websites that Kevin gave us. Then we put the knowledge we learned into a presentation and explained the concept with a fun restaurant analogy.
